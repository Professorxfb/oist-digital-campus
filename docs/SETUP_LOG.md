### Backend

The first Composer create-project attempt failed because the PHP CLI environment did not have the zip extension enabled and no `unzip` or `7z` command was available.

The source-based Composer retry failed because the Git clone exceeded Composer's 300-second process timeout, then Composer fell back to dist and hit the same missing zip/unzip/7z issue.

The Laravel project files were then created successfully by running Composer with the PHP zip extension loaded at runtime.

Dependency installation initially stopped because the PHP `fileinfo` extension was not enabled in the CLI environment.

The dependency installation completed successfully after running Composer with both `zip` and `fileinfo` loaded at runtime.

Important environment note:

- `php_zip.dll` exists but is not enabled by default in the active CLI `php.ini`.
- `php_fileinfo.dll` exists but is not enabled by default in the active CLI `php.ini`.
- Active CLI PHP configuration file: `D:\Software\php-8.5.7-nts-Win32-vs17-x64\php.ini`
